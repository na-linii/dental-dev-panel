3. Манифест (ключевые правила)

Принципы

  • + Код = инфраструктура, бизнес-логика = промпты + YAML
  • + Router = prompt + LLM + Command(goto=intent). 0 if-statements
  • + llm_input_messages для transient context (чистый checkpoint)
  • + 1 клиника = 1 YAML файл, код не меняется
  • + Все промпты в Langfuse, sync при старте

Запрещено

  • - Хардкод бизнес-логики (if greeting, if handoff)
  • - Модификация state["messages"] в hooks
  • - Intent remapping в router_node
  • - Heuristic handlers (regex для приветствий)
  • - Дублирование данных между файлами

6. Промпты (4 production)

dental-router 3 интента

faq (приветствия, info, жалобы, off-topic), booking (запись, отмена, перенос), confirm (ответ на напоминание при наличии [ИНФОРМАЦИЯ])

dental-booking 8 шагов

Услуга → Дата → get_availability → Показать → Выбор → Телефон/resolve → book_appointment → Завершение. {{booking_rules}} из YAML.

dental-faq Tier 1+2

Tier 1: YAML (адрес, цены, врачи). Tier 2: pgvector (подробности). handoff_to_human для жалоб. "Уточните по телефону" для оплаты.

dental-confirmation 3 tool

confirm_visit / decline_visit / reschedule_visit. Reschedule = cancel старой + новый booking flow через router.
